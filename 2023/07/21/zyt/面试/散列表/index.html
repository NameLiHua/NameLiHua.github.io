散列表中使用到了红黑树和链表

相当于数组的变种，相当于映射。

hash函数

key存入之前需要使用hash函数进行转换，实际上存的key并不是key本来的样子。

![[Pasted image 20230717162305.png]]

hash冲突

因为当前再好的hash函数都不能保证：key1 != key2 && hash(key1) != hash(key2)
当：**key1 != key2 && hash(key1) == hash(key2) *时，hash冲突就产生了。

解决办法：拉链法

拉链法就是：当产生hash冲突的时候，公用同一个key，同时将val成链表存储。
![[Pasted image 20230717163002.png]]
这个方法可能会导致散列表退化成链表。

所以引入红黑树至散列表中，将拉链法中的链表替换为红黑树，提高查找效率
